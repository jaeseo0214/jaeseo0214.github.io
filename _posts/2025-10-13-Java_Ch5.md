# Java의 예외 처리


프로그래밍을 실행하다 보면 오동작 또는 결과에 안 좋은 영향을 끼치는<br>
예상하지 못한 상황이 발생하기도 한다.<br>
이 상황 발생이 바로 '예외' 이다.<br>
만약 코드가 잘못 짜여져 있으면 컴파일러가 오류로 걸러주지만,<br>
'예외'는 사용자가 잘못된 입력을 하거나 배열 인덱스가 배열 크기보다 커지는 등,<br>
이런 사건들에 의해 프로그램 실행 중 발생한다.<br>


실행 중 예외가 발생하게 되면, 자바 플랫폼이 먼저 알게되어 현재 실행 중인<br>
자바 응용 프로그램에게 예외를 전달해 대응하게 한다.<br>
만약 예외에 대처하는 코드가 작성되어있지 않다면 응용 프로그램은 강제종료 된다.<br>


그렇다면 자바의 예외처리는 어떤 식으로 이루어질까?<br>


## 1. try-catch-finally 문


자바의 예외 처리는 아래와 같이 try-catch-finally 문을 사용한다.<br>


```java
try{
  예외 발생 가능성이 있는 실행문
}
catch (처리할 예외 타입 선언){
  예외 처리문
}
finally{
  예외 발생 여부와 상관없이 무조건 실행되는 문장 //생략 가능
}
```


예외 발생 가능성이 있는 실행문은 try {} 블록으로 묶고,<br>
예외 처리 코드는 catch {} 블록에 작성한다.<br>
주의할 점은 예외 처리 하나 당 catch {} 블록을 하나씩 작성해야 한다.<br>


try 블록에서 예외가 발생하면 catch 블록 > finally 블록 순서로,<br>
예외가 발생하지 않으면 try 블록 > finally 블록 순서로 실행된다.<br>
물론 발생한 예외와 처리할 예외 타입 선언이 같아야 한다.<br>
catch 블록 여러개를 선언해서 여러 타입의 예외를 처리하는 것도 가능하다.<br>






## 2. 자바의 예외 클래스


자바 플랫폼은 응용프로그램이 실행 중 오류를 탐지할 수 있도록<br>
많은 예외를 클래스 형태로 제공한다. 아래는 자주 활용되는 예외들이다.<br>


| 예외 타입 | 예외 발생 경우 | 패키지 |
| -------- | ------------- | ------ |
| ArithmeticException | 정수를 0으로 나눌 때 | java.lang |
| NullPointerException | null 래퍼런스를 참조할 때 | java.lang |
| ClassCastException | 변환할 수 없는 타입으로 객체를 변환할 때 | java.lang |
| OutOfMemoryError | 메모리가 부족한 경우 | java.lang |
| ArrayIndexOutOfBoundsException | 배열의 범위를 벗어난 접근을 할 때 | java.lang |
| IllegalArgumentException | 잘못된 인자를 전달할 때 | java.lang |
| IOException | 입출력 동작 실패, 인터럽트 시 | java.io |
| NumberFormatException | 문자열이 나타내는 숫자와 일치하지 않는 타입의 숫자로 변환할 때 | java.lang |
| InputMismatchException | Scanner 클래스의 nextInt() 호출 >>> 사용자가 정수가 아닌 'a'와 같은 문자를 입력할 때 | java.util |



간단한 예제 프로그램으로 예외 처리를 연습해 보겠다.<br>


```java
import java.util.InputMismatchException;
import java.util.Scanner;

public class prc {
	public static void main(String[] args) {
		Scanner sc = new Scanner(System.in);

		int a, b;
		int sum = 0;

		System.out.print("정수 2개를 입력하세요>>>");
		
		try {
			a = sc.nextInt();
			b = sc.nextInt();
			sum = a + b;
			System.out.println("합은 " + sum + "입니다.");
		}
		catch(InputMismatchException e) {
			System.out.println("정수를 입력하세요.");
		}
		sc.close();
	}
}
```


위 프로그램은 InputMismatchException 예외 타입을 사용한 예시이다.<br>
만약 사용자가 정수가 아닌 값을 입력을 시도하면 catch 블록으로 점프해<br>
예외 처리문을 출력해 준다.<br>


```java
import java.util.Scanner;

public class prc {
	public static void main(String[] args) {
		Scanner sc = new Scanner(System.in);

		int a, b;
		int result = 0;

		System.out.print("정수 2개를 입력하세요>>>");
		
		a = sc.nextInt();
		b = sc.nextInt();
		
		try {
			result = a / b;
			System.out.println("나눗셈 결과는 " + result + " 입니다.");
		}
		catch(ArithmeticException e) {
			System.out.println("0으로 나눌 수 없습니다.");
		}
		sc.close();
	}
}
```


위 프로그램은 ArithmeticException 예외 타입을 사용한 예시이다.<br>
만약 사용자가 정수를 0으로 나누려고 시도하면 catch 블록으로 점프해<br>
예외 처리문을 출력해 준다.
(여기선 사용하지 않았지만 위에서 사용한 InputMismatchException 예외도 함께 처리 가능하다!)<br>
