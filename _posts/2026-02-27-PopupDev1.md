# [개발 동기: "성수동 팝업스토어, 더 스마트하게 즐길 수는 없을까?"]


---


성수동은 명실상부 대한민국 팝업스토어의 성지이다.<br> 
하지만 직접 발로 뛰어보니 몇 가지 명확한 불편한 점을 발견할 수 있었다.<br>

1. 지리적 분산: 팝업스토어들이 연무장길, 서울숲 근처 등 넓은 지역에 흩어져 있어 동선이 꼬이기 쉬웠다.<br>

2. 시간의 불확실성: 각 스토어마다 대기 시간(웨이팅)과 운영 시간이 제각각이라 무작정 찾아갔다가는 낭패를 보기 일쑤였다.<br>

3. 체력과 효율: "방금 지나온 길인데 다시 돌아가야 하네?"라는 상황이 반복되어 즐거워야 할 나들이가 노동이 되어버렸다.<br>

"내가 가고 싶은 곳들만 쏙쏙 골라, 가장 짧고 효율적인 루트를 알려준다면?"<br>

이런 단순한 아이디어에서<br> 
'팝업스토어 최적 경로 계산 서비스' 기획을 시작하게 되었다.<br>


---





# [서비스 기획]


일단 개발 초보인 만큼,<br>
최소 기능 MVP를 정해서 구현하고,<br>
확장해 나가는 방식으로 진행하기로 했다.<br>


## 1. [서비스 정의]


* 목표: 여러 곳의 팝업스토어를 방문하려는 사용자에게 최적의 방문 순서와 거리를 계산하여 제공

* 핵심 가치: 위치 기반 알고리즘을 통한 이동 시간 단축 및 사용자 편의성 증대.


## 2. [핵심 기능]


* 데이터 : 팝업스토어 관리 -> DB에 저장된 팝업스토어의 이름, 위치(위경도) 데이터 관리.<br>

* 탐색 : 스토어 다중 선택 -> 사용자가 지도 또는 리스트에서 방문하고 싶은 팝업스토어를 여러 개 선택.<br>

* 알고리즘 : 최적 순서 계산	Greedy 알고리즘을 활용하여 현재 위치에서 가장 효율적인 방문 순서 도출.<br>

* 시각화: 인터랙티브 지도 카카오맵 API를 활용하여 마커(장소)와 폴리라인(경로)을 화면에 표시.
